    07 高度塌陷与BFC

    hasak 6/18/2022 CSS

    # 07 高度塌陷与BFC

    # 1.高度塌陷

    在浮动布局中，父元素的高度默认是被子元素撑开的

    • 当子元素浮动后，会完全脱离文档流，将会无法撑起父元素高度，导致父元素的高度丢失
    • 父元素高度丢失后，其下的元素会上移

    所以高度塌陷是浮动布局中比较常见的一个问题，这个问题我们必须要进行处理

    # 2. BFC

    **BFC（Block Formatting Context)**块级格式化环境

    • BFC是一个CSS中的隐含属性，可以为一个元素开启BFC
    • 开启BFC该元素会变成一个独立的布局区域

    元素开启BFC后的特点

    • ==不会被浮动元素覆盖==
    • ==可以包含浮动的元素==
    • ==父子元素外边距不会重叠==

    BFC的开启方式

    元素或属性 说明
    <html> 文档根元素
    float:left
    float:right
    		 浮动元素（flaot不为none）
    position:absolut
    position:fixed
    		 绝对定位元素
    overflow:hidden
    overflow:scroll
    overflow:auto    		 overflow不为visible和clip的块元素
    display:iniline-block 行内块元素
    display:table-cell 表格单元，默认值
    display:table-caption 表格标题，默认值
    display:table
    display:table_row
    display:table-row-group
    display:tabel-header-group
    display:table-footer-group
    display:inline-table    		 匿名的表格单元、分别为HTML表格、表行、
    表体、表头和表脚的默认值
    display:flow-root
    display:flex
    display:inline-flex的直接子元素    		 Flex项，如果他们本身既不是flex，也不是grid或table容器
    display:grid
    display:inline-grid的直接子元素    		 Grid项，如果他们本身既不是flex，也不是grid或table容器
    contain:layout
    contain:content
    contain:paint
    column-count不为auto
    column-width不为auto    		 Multicol容器，包含column-count：1
    column-span:all 总是创建一个新的格式化上下文，即column-span：all元素不在multicol容器中

    但是这些方式都会有一些缺陷。

    通过其他特殊方式开启元素的BFC

    • 设置为浮动（不推荐）：宽度变化，不再独占一行，下方元素被覆盖。总不能让所有的元素都浮动。

    • 设置为行内元素（不推荐）：宽度变化，不再独占一行，同时与下方元素产生一些空隙

    • 设置overflow为非visible值：保持了独占一方的特性（保持了宽度），与下方元素位置没变。
      • 常用的方式为元素设置overflow:hidden（overflow:auto也是可以的）开启其BFC，使其可以包含浮动元素
      • overflow:scroll，会出现滚动条，如果不需要滚动条则不推荐
      • overflow的方式也会出现一些问题，如不能完全清除div2布局上收到的影响

    总结：

    • 可以通过变成浮动元素，来防止自身被浮动元素覆盖
    • 可以设置行内块，来防止自身以及其他元素被浮动元素覆盖
    • 可以设置overflow属性，包含浮动元素

    # 3. clear

    clear作用：清除浮动元素对当前元素所产生的影响

    • 本质是为元素添加一个margin-top属性， 值由浏览器自动计算，下例添加了200px；

    可选值：

    • left清除左侧浮动元素对当前元素的影响
    • right 清除右侧浮动元素对当前元素的影响
    • both清除两侧浮动元素中影响较大一侧的元素怒对当前元素的影响
    div {
    font-size:50px;
}
.b1 {
    width:200px;
    height:200px;
    background-color:#bfa;
    float:left;
}
.b2 {
    width:200px;
    height:200px;
    background-color:orange;
    clear:both;
}
<div class="b1">1</div>
<div class="b2">3</div>

    # 4. after

    除了以上解决高度塌陷问题的方式，还有伪元素选择器::after

    • ::after伪元素是一个行内元素，需要转化为块元素，不然撑不起父元素高度。
    .box1 {
    border:10px solid red;
}
.box1::after{
    display:block;
    content:' ';
    clear:both;
}
.box2 {
    width:100px;
    height:100px;
    background-color:#bfa;
    float:left;
}

<div class="box1">

    # 5. clearfix

    垂直布局中，会出现边距重叠问题：相邻的垂直方向外边距会发生重叠现象

    • 子元素设置了一个margin-top之后，父元素会跟随子元素一起移动，此时margin相邻就会发生重叠

    可以通过伪元素解决，在子父元素之间我们使用before伪元素选择器，将两元素隔开

    • content必须得有内容，或者添加display:inline-block属性，这两种方式都会增加一个高度
    • 使用display:table可以解决问题
      • **display:table**也能开启BFC,开启BFC的一个特点就是父子元素外边距不会重叠
    .box1 {
    width:200px;
    height:200px;
    background-color:#bfa;
}
.box1::before{
    content:' ';
    display:table;
}
.box2 {
    width:100px;
    height:100px;
    background-color:orange;
    margin-top:100px;
}

<div class="box1">
	<div class="box2"></div>
</div>

    # 总结：

    • 高度塌陷问题，一般用::after
    • 外边距重叠问题，一般用::before

    既解决高度塌陷问题，又解决外边距重叠问题： clearfix

    • .clearfix::before 可以解决外边距重叠问题
    .clearfix::before{
    content: '';
    display:table;
}

    border:10px solid red;
}
.clearfix::before,
.clearfix::after {
    content:'';
    display:table;
    clear:both;
}
.box2 {
    width:100px;
    height:100px;
    background-color:#bfa;
    margin-top:100px;
}

	<div class="box2"></div>
