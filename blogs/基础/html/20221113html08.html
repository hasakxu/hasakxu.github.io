    # 08 定位

    **定位（position）**是一种更加高级的布局手段，通过定位可以将元素摆放到页面的任意位置

    使用position属性来设置定位

    可选值 含义
    static 不开启定位，元素是静止的，默认值
    relative 开启元素的相对定位
    absolute 开启元素的绝对定位
    fixed 开启元素的固定定位
    sticky 开启元素的粘滞定位

    # 1. 相对定位 relative

    当元素的position属性值设置为relative时，则开启了元素的相对定位

    • 需要设置偏移量来产生变化
    • 使用相对定位后，只会移动自身的布局位置，而不会对已存在的其他元素产生任何影响

    # 偏移量（offset）

    当元素开启相对定位后，可以通过偏移量来设置元素的位置

    offset属性 含义
    top 定位元素和定位位置的上边距离
    bottom 定位元素和定位位置的下边距离
    left 定位元素和定位位置的左边距离
    right 定位元素和定位位置的右边距离

    **top**和 **bottom**两个属性定位元素垂直方向的位置

    • **top**值越大，定位元素越靠下
    • **bottom**值越大，定位元素越靠上

    **left**和 **right**两个属性定位元素垂直方向的位置

    • **left**值越大，定位元素越靠右
    • **right**值越大，定位元素越靠左
    body {

    # 特点：

    1. 元素开启相对定位后，如果不设置偏移量元素，元素不会发生任何变化（这里不仅仅是位置）
    2. 相对定位是==参照于元素在文档流的原始位置进行定位的==
    3. 相对定位会==提升元素的层级==（表现为可以覆盖其他元素）
    4. 相对定位==不会改变元素的性质==（即块元素还是块元素，行内还是行内）
    .box2 {
    width:200px;
    height:200px;
    background-color:orange;
    position:relative;
    bottom:200px;
    left:100px;

    Q1：多个div都设置相对定位，他们的层级关系会怎么样？谁会被谁覆盖？

    ​ 在没有设置层级或层级z-index设置相同值时，优先显示靠下的元素。

    Q2:相对定位的第三个特点会提升元素的层级，是否就类似于浮动一样脱离了文档流

    ​ 浮动脱离文档流，不再占据页面位置；相对定位仍然占据页面位置。

    参考浮动和相对定位的区别：

    • 参考系不同：浮动的参考系时其父元素；相对定位是相对于自身
    • 可移动方向不同：浮动只能左右移动；相对定位可以上下左右移动
    • 影响不同：浮动会影响页面布局（包括下方元素位置影响和高度塌陷问题）；相对定位不影响页面布局
    • 性质不同：浮动会改变元素的性质（使得不再独占一行，宽高都会被内容撑开）；相对定位则不改变元素性质
    • 文字环绕：浮动不会覆盖文字；相对定位可以覆盖文字
    • 此外，==浮动和相对定位都不会从父元素中移出==

    Q3:相对定位不会改变元素的性质，但原始位置是否不再独占一行

    ​ 其原始位置依然独占一行

    Q4:相对定位的第四个特点意味着行内元素也可以使用相对定位吗？

    ​ 是的

    # 2. 绝对定位

    当元素的position属性设置为absolute时，则开启了元素的绝对属性

    # 特点

    1. 开启了绝对定位后，如果不设置偏移值，元素的位置不会发生变化（只是位置不变）
    2. 绝对定位是==相对于其他包含块进行定位的==
    3. 绝对定位会，元素会提升一个等级
    4. 开启绝对定位后，元素会从文档流中
    5. 绝对定位会==改变元素的性质：行内变块，块的宽高默认被内容撑开==
    body {
    width:200px;
    height:200px;
    background-color:#bfa;
}
.box2 {
    background-color:orange;
    position:absolute;
}
.box3 {
    width:300px;
    height:300px;
    background-color:yellow;
}

    # 水平方向的布局

    之前写过，水平方向的布局等式：

    margin-left + border-left + padding-left + width + padding-right + border-right + margin-right = 其父元素的宽度

    当使用绝对定位时，添加left 和 right 两个值。

    left + margin-left + border-left + padding-left + width + padding-right + border-right + margin-right + right = 其父元素的宽度

    当发生过度约束时

    • 如果9个值中没有auto，则自动调整right值使得等式满足
    • 如果9个值中有auto，则自动调整auto的值使得等式满足
      • 可设置auto的值： margin-left margin-right width left right
      • left 和right的值默认是auto,如果没有设置，会自动调整这俩值让等式成立，优先调right

    水平居中

    <style>
    .box1 {
        width:500px;
        height:500px;
        background-color:#bfa;
        position:relative;
    }
    
    .box2 {
        width:100px;
        height:100px;
        background-color:orange;
        
        margin-left:auto;
        margin-right:auto;
        position:absolute;
        left:0;
        right:0;
    }
</style>

    <div class="box2">
        
    </div>

    # 小结

    • 水平方向的布局等式：

      margin-left + border-left + padding-left + width + padding-right + border-right + margin-right = 其父元素的宽度

    • 垂直方向布局的等式：

      top + margin-top + border-top + padding-top + height + padding-bottom + border-bottom + margin-bottom + bottom = 其父元素的高度

    • 上述等式的过度约束规则盒模型要保持一致，只是在没有auto时，会自动调整 top bottom left right

    # 3. 固定定位

    将元素的position 属性改为 fixed, 则开启了元素的固定定位

    # 特点

    • 固定定位也是一种绝对定位，所以大部分特点和绝对定位一样
    • 唯一不同的是：==固定定位永远参照浏览器的视口（viewport)进行定位，不随网页的滚动条滚动==

    # 4. 粘滞定位

    将元素的position属性设置为sticky，则开启了元素的固定定位，兼容性不好

    # 特点：

    • 元素到达某一个位置后固定
      • 该元素根据文档流进行定位，即相对于包含块进行偏移
      • 偏移量不会影响任何其他元素的位置
      • 粘性元素总是”粘“到其最近的具有”滚动机制“的祖先元素 （当overflow为hidden、scroll、auto、overlay时创建，即使该祖先不是最近的实际滚动祖先

    # 5. 几种定位的对比

    定位方式 是否不设置偏移量，
    元素不会发生改变    		 是否脱离文档流 是否改变元素性质 是否提升元素层级 参考系
    relative（相对定位） ✔ ❌ ❌ ✔ 参照于元素在文档流中的位置
    absolute（绝对定位） ❌ ✔ ✔ ✔ 参照于其包含块
    fixed（固定定位） ❌ ✔ ✔ ✔ 参照于浏览器的视口
    sticky(粘滞定位) ❌ ✔ ✔ ✔ 参照于浏览器的视口

    # 6. 补充：元素层级

    对于开启了定位元素，可以通过z-index属性来指定元素的层级

    • z-index需要一个整数作为参数，值越大，元素的层级越高；元素的层级越高，越优先显示
    • 如果元素的层级一样，则优先显示靠下的元素
    • 祖先的元素层级再高，也不会盖住后代元素

    Q:浮动也有层级概念吗？如果有，浮动和定位的层级关系是什么样的？

    ​ 会出现以下的现象

    • 给float设置z-index多大都没用，还是会被覆盖
    • 没有设置z-index或设置的z-index大小 ≥ 0时，浮动层级没有定位的层级高
    • 设置z-index < 0 时，浮动层级可以定位的层级高

    # 总结

    一般情况下：

    • 页面的整体结构大多采用浮动、块进行布局
    • 页面某些模块结构一般采用定位进行微调
